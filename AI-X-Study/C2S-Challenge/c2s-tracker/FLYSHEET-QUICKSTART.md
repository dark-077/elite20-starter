## 如何添加第一条记录

### 例子：为"论文理解"添加一条记录

假设您正在读一篇论文，需要记录对论文的理解：

1. 打开 **论文理解追踪** 数据表
2. 点击底部的 **"+"** 或 **"+ 添加记录"**
3. 填写字段：

| 字段 | 填写内容 |
|------|----------|
| 名称 | `《共生智能》论文 - 摘要理解` |
| 类型 | `摘要` |
| 状态 | `进行中` |
| 完成度 | `30%` |
| 质量等级 | `待改进` |
| 论文页码 | `P.1-2` |
| 个人解读 | `这篇论文提出了共生智能的概念，强调人与AI的互补关系...` |

## 日常使用流程

### 每日工作流

```
1. 打开飞书 C2S 产出物追踪
2. 查看今日任务（根据状态筛选）
3. 完成一项工作后更新状态
4. 记录关键AI协作（特别是AAR）
```

### 状态流转

| 产物 | 状态流转 |
|------|----------|
| 论文理解 | 未开始 → 进行中 → 待审核 → 已完成 |
| KSTAR | 未开始 → K阶段 → S阶段 → T阶段 → A阶段 → R阶段 → 已完成 |
| 个人联结 | 未开始 → 构思中 → 初稿 → 完善中 → 已完成 |
| AAR | 未开始 → 时间线整理中 → 分析中 → 初稿 → 已完成 |

## 协作轨迹记录方法

每次与AI协作后，立即在 AAR 复盘追踪 中记录：

### 快速记录模板

```
AI模型: [选择的模型]
对话轮次: [数字]
关键提示词: [粘贴你的提示词]
AI输出摘要: [1-2句话总结AI说了什么]
协作质量: [低/中/高/优秀]
对话时间: [今天的日期]
```

### 示例

```
AI模型: ChatGPT-4o
对话轮次: 3
关键提示词: "请帮我分析这篇论文的核心观点，用中文总结3个要点"
AI输出摘要: "论文提出了共生智能的概念，强调人机互补..."
协作质量: 高
对话时间: 2026-04-30
```

## 检查清单使用

每个字段下方都有对应的检查清单，每周至少自检一次：

### 论文理解自检
- [ ] 是否有原文引用？（页码标注）
- [ ] 是否有个人解读（非纯复述）？
- [ ] 是否区分了'论文讲的'和'我讲的'？

### KSTAR自检
- [ ] K - 是否明确了需要的具体信息？
- [ ] S - 是否有具体的时间/资源约束？
- [ ] T - 任务是否具体可衡量？
- [ ] A - 是否有具体的行动步骤？
- [ ] R - 结果是否有可衡量的指标？

### 个人联结自检
- [ ] 是否提及了具体的专业背景？
- [ ] 是否有真实发生的事件（非假设）？
- [ ] 是否避免了泛泛而谈（如'我很感兴趣'）？

### AAR自检
- [ ] 是否有具体的时间戳事件？
- [ ] 是否有具体的数据或事实？
- [ ] AI协作记录是否完整？
- [ ] 协作轨迹是否记录了AI模型和对话轮次？

## 常见问题

### Q: 多个数据表之间需要关联吗？
A: 不需要。每个数据表独立追踪一个产物。它们通过同一个多维表格组织在一起。

### Q: 完成度是怎么计算的？
A: 手动填写。您自己评估当前完成百分比。这是给自己看的进度追踪。

### Q: 检查清单在哪里？
A: 检查清单是参考标准，不需要录入系统。您可以打印出来，每周自检时对照。

### Q: 如何分享给其他人？
A: 点击右上角 **"分享"** 按钮，可以邀请同事或设置链接访问权限。
